PROJETO DE TESTES QA

OBJETIVO
Este projeto foi criado para simular atividades reais de QA, incluindo testes manuais, reporte de bugs e testes de API.

SISTEMAS TESTADOS
- Aplicação web: https://www.saucedemo.com
- API: Reqres (https://reqres.in)

ATIVIDADES DE TESTE
- Criação de casos de teste manuais
- Execução de testes funcionais
- Reporte e documentação de bugs
- Testes de API utilizando Postman

FERRAMENTAS UTILIZADAS
- Postman
- GitHub
- DevTools do navegador
- Jira

ESTRUTURA DO PROJETO
- test-cases/: Casos de teste e cenários manuais
- bug-reports/: Bugs documentados com detalhes
- api-tests/: Coleções e scripts do Postman
- evidence/: Evidências (prints e respostas de API)

COMO EXECUTAR OS TESTES DE API
1. Importar a collection no Postman
2. Selecionar a coleção de requisições
3. Executar manualmente ou pelo Collection Runner
4. Validar os resultados na aba "Tests"

PRINCIPAIS APRENDIZADOS
- Escrita de casos de teste estruturados e claros
- Identificação, reprodução e documentação de bugs
- Validação de respostas e status code de APIs
- Organização profissional de documentação de QA
