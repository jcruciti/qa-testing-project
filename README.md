QA TESTING PROJECT

OBJECTIVE
This project simulates a real-world QA workflow by applying structured testing practices on a web application and a public API, focusing on defect identification, validation, and documentation.

SYSTEMS UNDER TEXT
- Web Application: https://www.saucedemo.com
- API: https://reqres.in

SCOPE OF TESTING
The testing scope includes validation of core user flows and API endpoints:
- User authentication (valid and invalid scenarios)
- Product selection and cart functionality
- Checkout process
- API endpoints validation (GET, POST)

TEST ACTIVITIES
- Designed and executed X+ manual test cases covering positive and negative scenarios
- Performed functional and exploratory testing to identify unexpected behaviors
- Reported X bugs with detailed reproduction steps, expected vs actual results, and evidence
- Executed API tests using Postman, validating:
    - Status codes
    - Response structure
    - Data consistency

TOOLS USED
- Postman
- GitHub
- Browser DevTools
- Jira

PROJECT STRUCTURE
- test-cases/ → Structured test cases with clear steps and expected results
- bug-reports/ → Documented defects with evidence and reproducibility steps
- api-tests/ → Postman collections and test scripts
- evidence/ → Screenshots and API response validation

TEST STRATEGY
The testing approach was based on:
- Functional testing of critical user journeys
- Negative testing to validate system robustness
- Exploratory testing to uncover unexpected issues
- API validation to ensure backend reliability
  
KEY LEARNINGS
- Improved ability to design structured and maintainable test cases
- Gained experience in bug reporting aligned with industry standards
- Strengthened API testing skills using Postman
- Learned how to organize QA artifacts in a scalable and professional way
