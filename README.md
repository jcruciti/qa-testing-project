QA TESTING PROJECT

OBJECTIVE  
This project was created to simulate real QA activities, including manual testing, bug reporting, and API testing.

SYSTEMS UNDER TEST  
- Web application: https://www.saucedemo.com  
- API: Reqres (https://reqres.in)

TESTING ACTIVITIES  
- Creation of manual test cases  
- Execution of functional tests  
- Bug reporting and documentation  
- API testing using Postman  

TOOLS USED  
- Postman  
- GitHub  
- Browser DevTools  
- (Optional) Jira  

PROJECT STRUCTURE  
- test-cases/: Test cases and manual scenarios  
- bug-reports/: Documented bugs with details  
- api-tests/: Postman collections and scripts  
- evidence/: Evidence (screenshots and API responses)  

HOW TO RUN API TESTS  
1. Import the collection into Postman  
2. Select the request collection  
3. Run manually or via Collection Runner  
4. Validate results in the "Tests" tab  

KEY LEARNINGS  
- Writing structured and clear test cases  
- Identifying, reproducing, and documenting bugs  
- Validating API responses and status codes  
- Professional organization of QA documentation
